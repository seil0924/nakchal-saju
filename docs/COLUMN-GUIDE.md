# 사주 칼럼(/column) 글쓰기 가이드

자체 블로그는 `content/column/` 폴더의 마크다운(`.md`) 파일로 굴러갑니다.
**파일 하나 = 글 하나**, 파일명이 URL이 됩니다.

## 새 글 추가하는 법

1. `content/column/` 에 `<slug>.md` 파일 생성 (slug = URL, 영문·숫자·하이픈 권장)
   - 예: `balju-gunghap.md` → `nakchal-saju.vercel.app/column/balju-gunghap`
2. 맨 위에 frontmatter(메타정보) 작성 후 본문
3. 커밋 → Vercel 자동 배포. 끝.

## frontmatter 형식

```markdown
---
title: "글 제목 (검색 결과에 그대로 노출됨)"
description: "한 줄 요약. 150자 이내. 구글·SNS 미리보기·목록 카드에 쓰임"
date: "2026-07-21"          # YYYY-MM-DD
tags: ["입찰", "사주", "사정률"]
cover: "/column/xxx.jpg"    # (선택) 대표 이미지. 없으면 기본 OG 이미지
draft: false               # (선택) true면 목록·사이트맵에서 숨김
---

본문 마크다운...
```

## 예약발행 / 초안

- **초안 숨기기**: `draft: true` → 사이트에 안 뜸
- **예약발행**: `date`를 미래로 → 그 날짜까지 숨겨짐
  - ⚠️ 단, Vercel은 자동 재빌드를 안 하므로 **발행일에 재배포(또는 재배포 훅)** 가 필요.
    완전 자동 예약이 필요하면 Vercel Cron/Deploy Hook을 별도로 걸어야 함.

## 마크다운 볼드 주의점 (한글 조사)

CommonMark 규칙상 **볼드 끝이 따옴표·괄호로 끝나고 바로 한글 조사가 붙으면 볼드가 안 걸립니다.**

```
✗ 안 됨:  **'그래, 이 날'**은 확신      →  그대로 별표가 보임
✗ 안 됨:  그 **운칠(運七)**을            →  그대로 별표가 보임
✓ 됨:    '**그래, 이 날**'은 확신        →  따옴표를 볼드 밖으로
✓ 됨:    그 **운칠**(運七)을             →  괄호를 볼드 밖으로
```

요령: **볼드는 한글 글자에서 끝나게** 하고, 따옴표·괄호·문장부호는 볼드 바깥으로 뺀다.

## SEO 자동 처리 (신경 안 써도 됨)

글을 올리면 아래가 자동으로 적용됩니다.
- `<title>`·메타 description·OG/트위터 카드
- Article 구조화 데이터(JSON-LD) — 구글이 발행일·제목 인식
- `sitemap.xml` 에 자동 등재 (발행일 = lastModified)
- canonical URL

## 네이버 병행 전략

1. **자체 칼럼에 먼저** 전체 글 발행 (구글이 원본으로 인식)
2. 며칠 뒤 **네이버 블로그엔 축약본** + "전체 보기 → 사이트 링크"
   (똑같이 복붙하면 구글이 중복 콘텐츠로 자체 블로그를 밀어냄)

## 낙찰사주 칼럼 문체 플레이북 (네이버 톤 기준)

실제 네이버 블로그 톤을 자체 칼럼에도 동일하게 적용한다. 핵심은 **명리(命理) 느낌**.

### 목소리
- **대표님 존댓말**, 감성적 통점 오프닝 → 명리 재구성 → 위로 → 무료 CTA
- 첫 문단은 밤잠·마감 전날·소수점 둘째자리 같은 **입찰 현장의 구체적 장면**으로
- 예측·보장 금지. "흐름으로 바꿔주는 것", "참고" 톤 유지

### 한자 병기 (사주 느낌의 핵심)
자주 쓰는 것: 落札四柱 · 運七技三 · 四柱命理 · 萬歲曆 · 節氣 · 眞太陽時 · 夜子時 · 日辰 · 命式 · 財/官 · 沖/刑/破 · 大運/歲運 · 時辰 · 會社
- 한글 뒤 괄호로: `일진(日辰)`, `대운(大運)`, `충(沖)·형(刑)·파(破)`
- **볼드는 한글에서 끝내고 괄호는 밖으로**: `**운칠**(運七)을` (O) / `**운칠(運七)**을` (X)

### 재사용 HTML 블록 (마크다운 안에 그대로 붙여넣기)

**굵은 pull-quote** — 대표 속마음 한 줄:
```markdown
> **"이번 건, 넣는 게 맞나."**
```

**다크 계산 배너** — 신뢰(점 아니라 계산):
```html
<div class="calcrow">
  <div><b>節氣</b><span>절기를 천문으로 판정</span></div>
  <div><b>眞太陽時</b><span>진태양시 −30분 보정</span></div>
  <div><b>夜子時</b><span>서머타임·야자시 자동 보정</span></div>
</div>
```

**판결 인용구** — 오늘의 사정률 신호 (다크 골드박스):
```html
<blockquote class="verdict">"오늘은 기운이 눌리는 날 —<br>서두르기보다 관망이 유리한 흐름입니다."<cite>— 낙찰사주 · 오늘의 사정률 신호</cite></blockquote>
```

**면책 ※** — 글 맨 끝:
```html
<p class="disc">※ 낙찰사주는 사주명리(四柱命理) 이론에 기반한 참고·오락 정보입니다. 낙찰 여부나 특정 결과를 예측·보장하지 않으며, 실제 투찰 금액 산정의 근거로 사용할 수 없습니다.</p>
```

> ⚠️ HTML 블록은 앞뒤로 **빈 줄**을 둬야 렌더된다. HTML은 신뢰된 저자(운영자)만 작성 — 외부 사용자 마크다운을 그대로 넣지 말 것(XSS).
